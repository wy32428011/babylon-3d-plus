# 场景级模型生成器设计

## 目标

保留 POI 库“模型生成器”以及点击/拖拽创建能力。生成器实体在编辑态仍有可拾取标记和 Transform，方便用户把配置入口放在场景任意位置；但该 Transform 只控制配置标记，绝不作为任何自动货物的生成坐标。

一个场景以 `entityIds` 中第一个模型生成器为全局自动模型管理器，统一决定普通输送机、普通堆垛机和 `warehouseFlow` 运行时货物使用的模板。货物的存在、移动、落位和销毁仍由各自设备遥测及仓储状态机负责。

## 核心语义

### 1. POI 与运行时彻底分离

- `components.modelGenerator` 继续保存在普通实体上，兼容现有场景文件和 Inspector。
- 实体 `transform` 只应用到青色线框标记。
- 生成器的运行时货物使用独立、无父级的货物根节点；位置来自输送面、货叉支撑点、目标 locator 或仓储协调器。
- 自动货物不可拾取、不进入 Hierarchy、不写入场景文件或撤销历史。

### 2. 单例策略

- 新建场景只允许创建一个模型生成器。再次点击或拖入时选中已有生成器，并记录提示，不创建副本。
- 复制、粘贴和阵列不得产生第二个模型生成器。
- 为兼容旧场景，运行时若发现多个生成器，按 `scene.entityIds` 顺序只启用第一个，其余仅显示编辑标记，并记录一次诊断。

### 3. 模板与规则

- 设备遥测决定是否需要货物实例；生成器只决定该实例使用哪种可视模板。
- 每个有效货物快照独立解析规则，因此不同设备和条码可以同时生成不同模型。
- 规则按顺序匹配。属性名先支持 `sourceId`、`deviceType`、`assetCode`，其余名称读取 `snapshot.fields`。
- 命中规则后使用 `rule.target ?? defaultTarget`；命中但没有可用目标时继续后续规则。
- 没有规则命中时使用 `defaultTarget`。
- 场景没有生成器、生成器没有可用模板或导入模型加载失败且无共享模板可回退时，普通输送机和堆垛机继续使用原有默认 Box，保证旧场景可视化不退化。
- `warehouseFlow` 没有可用模板时保持当前 fail-closed 行为，不创建仓储货物。

### 4. 并发与生命周期

- 普通输送机按 `assetCode + containerCode` 管理货物；普通堆垛机沿用相同唯一键和已落位 locator 状态。
- 每个货物拥有独立输出宿主、加载 token、目标签名、失败缓存和运行时脚本上下文。
- 模板规则变化时按当前设备或库位支撑点重新创建并替换可视输出，不改变设备/库位决定的位置。
- 导入模型异步加载完成前保留空货物根节点；过期加载结果立即释放。
- 停止预览、货物信号消失、设备实例删除、生成器配置切换、场景切换和 Runtime 销毁时统一释放 Mesh、材质、模型容器、贴图和外置脚本。

### 5. 坐标契约

- 所有生成输出的根节点表示货物底部支撑点。
- 内置 Mesh 子节点按自身高度向上偏移；导入模型继续使用底部中心归一化。
- 普通输送机和堆垛机把原默认 Box 中心坐标换算为支撑点，确保启用生成器前后默认 Box 位置不漂移。
- locator 落位统一使用 locator 盒体底面支撑点。

## 编辑器表现

- Project/POI 库保留“模型生成器”卡片和拖拽载荷。
- Inspector 保留名称、Transform 和完整模型生成器配置，并增加醒目说明：“标记位置仅用于编辑配置，不影响自动模型位置”。
- 生成器输出不参与选中、高亮、Gizmo 或阵列包围盒；场景聚焦仍只聚焦配置标记。

## 验收标准

- 拖动模型生成器标记后，普通输送机、堆垛机和仓储流货物位置不发生变化。
- 一个场景不能通过点击、拖拽、复制、粘贴或阵列创建第二个有效生成器。
- 同一时刻多个设备/条码可生成独立模型，规则可按设备身份或字段选择不同模板。
- 无生成器或无模板时普通设备仍显示原默认 Box；warehouseFlow 继续在无模板时拒绝启动。
- 停止预览后所有自动货物资源释放，编辑态仅显示 POI 标记。
- `npm run typecheck`、`npm run build`、`git diff --check` 通过。
