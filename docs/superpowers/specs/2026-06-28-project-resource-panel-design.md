# Project 资源库面板样式设计

## 背景

当前底部 `Project` 面板仍以 `Project` 标题、`扫描资源目录` 按钮和资产列表为主视觉。用户希望先把该区域调整为参考图中的资源库浏览器样式，暂不实现真实资源分类、搜索、导入或打开功能。

## 目标

- 将底部 Project 区域改造成资源库外观。
- 展示七个库页签：模型库、POI库、主题库、组合库、环境库、图表库、图片库。
- 保留“页签栏 + 搜索/筛选行 + 横向资源卡片区”的完整视觉结构。
- 当前只做样式与占位交互，不接真实资源功能。

## 非目标

- 不实现真实资源目录扫描入口的新交互。
- 不实现搜索过滤逻辑。
- 不实现模型、POI、主题、组合、环境、图表、图片的真实数据加载。
- 不改变 Scene、Hierarchy、Inspector、Toolbar 或 Babylon 运行时逻辑。

## 推荐方案

采用“单组件静态视觉壳”方案：在现有 `src/editor/panels/ProjectPanel.tsx` 中集中完成资源库页签、搜索栏和占位卡片展示。该方案改动集中、风险低，符合“先改样式，具体功能暂时不做”的边界。

## 界面结构

底部资源库面板由三层组成：

1. **资源库页签栏**
   - 横向展示 `模型库 / POI库 / 主题库 / 组合库 / 环境库 / 图表库 / 图片库`。
   - 默认选中 `模型库`。
   - 激活页签使用更亮的深色背景和科技蓝顶部强调线。

2. **筛选占位行**
   - 显示当前库对应的字段标签，例如 `模型名称`、`POI名称`。
   - 输入框使用占位文本，例如 `请输入模型名称...`。
   - 输入框仅作视觉占位，不触发过滤逻辑。

3. **资源卡片横向列表**
   - 使用深色卡片背景。
   - 缩略图区域使用科技蓝线框/图标式占位视觉。
   - 底部显示资源占位名称。
   - 内容横向排列，超出宽度时允许横向滚动。

旧的裸露 `Project` 标题和 `扫描资源目录` 按钮不再作为主视觉出现，避免和参考图样式冲突。

## 组件与状态设计

`ProjectPanel` 内部使用静态配置描述资源库：

- `key`：库的内部标识，例如 `model`、`poi`、`theme`。
- `label`：页签展示名称，例如 `模型库`。
- `searchLabel`：筛选行标签，例如 `模型名称`。
- `searchPlaceholder`：输入框占位文本。
- `items`：当前库的占位资源卡片。

组件使用 `useState` 保存当前选中的库页签。点击页签只切换激活态、筛选标签和占位卡片，不调用资源扫描、导入、加载场景等真实功能。

## 样式设计

继续复用项目已有暗色编辑器风格，并新增 Project 面板专属 class，让资源库样式和其他面板隔离。重点样式包括：

- 页签栏：深色背景、横向排列、激活态高亮。
- 筛选行：紧凑高度、深色输入框、低对比度标签。
- 资源卡片：固定尺寸、边框、深色底、科技蓝缩略图占位。
- 内容区：横向滚动，避免卡片挤压布局。

## 数据流

当前数据流保持简单：

1. 初始渲染默认读取 `模型库` 静态配置。
2. 用户点击其他页签。
3. `activeLibraryKey` 更新。
4. 面板根据当前库配置重新渲染筛选行与占位卡片。

不读取真实 Assets 目录，不改变编辑器 store，不影响已有场景数据。

## 验证标准

- 底部 Project 区域不再显示旧的 `Project + 扫描资源目录` 主视觉。
- 页面能看到七个资源库页签，且默认激活 `模型库`。
- 点击各页签后，激活态、筛选标签和卡片占位内容会切换。
- 搜索输入框只呈现占位文本，不执行过滤。
- 资源卡片横向排列，宽度不足时可以横向滚动。
- README 记录本次资源库样式改造与功能边界。

## 后续扩展

后续真正接功能时，可以将静态 `items` 替换为真实资源数据映射，并逐步恢复或重构资源扫描入口。真实功能接入应单独设计，避免与本次样式改造混在一起。
