# 底部图库区域高度调整设计

## 背景

当前底部 `Project` 资源库面板已经切换为资源库浏览器外观，包含页签栏、筛选占位行和横向资源卡片列表。但编辑器整体布局仍给底部区域固定 `180px` 高度，而资源库内部内容完整展示至少需要约 `210px`。因此在当前窗口中，图库区域占比偏小，资源卡片底部内容和留白展示不完整。

## 目标

- 增大底部图库区域占比，让资源库卡片、名称和底部空间完整可见。
- 保持当前 Unity-like 五面板布局结构不变。
- 优先采用简单、稳定、低风险的布局调整。
- 避免引入新的资源库业务逻辑或拖拽布局状态。

## 非目标

- 不实现 Scene 与图库之间的可拖拽分隔条。
- 不接入真实资源扫描、搜索、导入、拖拽或打开功能。
- 不重构 `ProjectPanel` 的静态资源配置。
- 不改变 Scene View、Hierarchy、Inspector、Toolbar、Console 或 Babylon 运行时逻辑。

## 选定方案

采用“固定加高到底部图库约 `260px`”方案：将编辑器根布局的底部行高从 `180px` 调整为 `260px`。

该方案比 `220px` 和 `240px` 更宽裕，符合用户反馈中“图库区域可以占比多一些，没有展示全”的核心诉求。虽然会让中央工作区高度减少约 `80px`，但在当前最小窗口高度 `720px` 下，中部工作区仍保留约 `412px` 高度，足够继续承载 Scene、Console、Hierarchy 和 Inspector 的基础展示。

## 布局结构

当前 `EditorLayout` 顶层结构为三行 CSS Grid：

1. 顶部工具栏：`48px`
2. 中央工作区：`minmax(0, 1fr)`
3. 底部资源库：`180px`

调整后变为：

1. 顶部工具栏：`48px`
2. 中央工作区：`minmax(0, 1fr)`
3. 底部资源库：`260px`

中央工作区仍使用原有三列结构：左侧 Hierarchy、中间 Scene + Console、右侧 Inspector。底部 `ProjectPanel` 仍独立位于 `bottomBar`，不会嵌入中央工作区，也不会改变其它面板的组件边界。

## 组件与样式设计

本次只需要调整布局层样式：

- 文件：`src/editor/layout/EditorLayout.module.css`
- 选择器：`.editorShell`
- 改动点：`grid-template-rows` 的第三行高度

`ProjectPanel` 内部样式维持现状：

- 页签栏高度仍为 `32px`。
- 筛选行高度仍为 `42px`。
- 卡片仍为 `112px` 高。
- 卡片列表仍横向排列，宽度不足时横向滚动。

通过增加外层底部区域高度，而不是压缩卡片或隐藏筛选行，可以保留当前资源库外观的完整视觉层级。

## 数据流

本次调整不改变数据流。

1. `EditorLayout` 渲染三段式编辑器壳。
2. CSS Grid 为 `bottomBar` 分配 `260px` 高度。
3. `ProjectPanel` 在更高的底部区域中按原有静态配置渲染页签、筛选行和资源卡片。
4. 用户切换资源库页签时，仍只更新 `ProjectPanel` 内部 `activeLibraryKey` 状态。

不读取真实 Assets 目录，不改变编辑器 store，不影响场景保存、加载、撤销或 Babylon 运行时同步。

## 边界与风险

- 中央 Scene 视口会因底部区域增大而减少高度，这是本方案的主要取舍。
- 在当前 `body` 最小高度 `720px` 下，布局仍有足够空间展示中央工作区。
- 如果未来资源卡片尺寸继续增大，固定高度可能再次不足；届时应单独设计可拖拽分隔条或 CSS 变量化布局。
- 当前方案不保存用户布局偏好，因此不会引入持久化兼容问题。

## 验证标准

- 底部 Project 资源库区域高度明显增加。
- 模型库资源卡片、资源名称和底部空间完整可见，不再被 `bottomBar` 裁切。
- 七个资源库页签和筛选占位行仍保持原有样式。
- Scene、Hierarchy、Inspector、Console 仍正常显示。
- README 记录本次图库区域高度调整。

## 后续扩展

如果后续用户需要在不同工作场景中动态分配空间，可单独设计 Scene 与 Project 之间的可拖拽分隔条，并考虑将高度偏好持久化到本地配置。该能力不纳入本次改动范围。
