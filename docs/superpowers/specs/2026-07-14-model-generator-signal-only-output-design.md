# 模型生成器仅信号触发输出设计

## 目标

模型生成器中的模型只作为运行时生成模板。用户把普通模型或内置基础网格拖入属性面板后，编辑态不直接显示该模型；只有 MQTT 信号命中生成规则时才创建派生输出，信号失效后立即销毁。

## 兼容策略

- 保留现有 ModelGeneratorComponent 数据结构和场景文件版本 1。
- defaultTarget 字段改为“共享生成模板”，不再代表编辑态默认显示或无匹配回退输出。
- rule.target 继续作为规则专属覆盖模型；规则未配置专属模型时使用 defaultTarget。
- 旧场景无需迁移。旧场景中的 defaultTarget 将自动按共享模板新语义运行。

## 编辑态行为

- 生成器始终只显示青色可拾取线框标记。
- defaultTarget 和 rule.target 都只是配置快照，不创建 Babylon Mesh、模型容器或脚本实例。
- Gizmo、显隐、锁定、聚焦和包围盒继续以稳定 TransformNode 根节点及线框标记为准。

## 运行态行为

1. 按完整 sourceId、deviceType、assetCode 绑定读取 MQTT 快照。
2. 排除超过 metadataTtlSeconds 的快照。
3. 选择 receivedAt 最新快照；时间相同保留绑定顺序靠前者。
4. 按规则顺序查找第一条属性名和值命中且具有有效目标的规则。
5. 命中规则后，先计算候选目标 rule.target ?? defaultTarget；候选目标为空属于不完整规则，忽略并继续后续规则。
6. 扫描后无有效命中目标、没有有效快照、没有规则命中或 TTL 过期时，当前输出必须销毁并保持为空。
7. 规则专属模型加载失败时，可在同一有效信号下回退共享模板；共享模板也失败时保持无输出。

## 输送线示例

- 共享生成模板：拖入一个货物模型。
- 规则一：front_has_goods 等于 true，规则覆盖模型可留空。
- 规则二：back_has_goods 等于 true，规则覆盖模型可留空。
- 任一有效规则命中时，在该生成器位置生成一个货物模型；两个字段均不命中、遥测过期或共享模板为空时销毁货物。

## Inspector 文案

- 默认生成对象改为“共享生成模板”。
- 规则生成对象改为“规则覆盖模型”，并标注可选。
- 空规则提示改为“暂无生成规则，运行态不会生成模型”。
- TTL 提示改为“数据超时后销毁当前生成模型”。

## 验证标准

- 配置共享模板后，编辑态仍只显示青色线框。
- 进入运行预览且无有效信号时不显示派生模型。
- front_has_goods 或 back_has_goods 命中时生成模板模型。
- 字段恢复、不匹配、MQTT 断开或 TTL 到期时销毁模型。
- 停止运行预览后只恢复编辑态线框，不显示模板模型。
- typecheck、build 和 git diff --check 通过。
